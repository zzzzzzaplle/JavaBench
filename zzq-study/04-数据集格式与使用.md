# 数据集格式与使用

## 一、数据集概述

JavaBench 提供三种上下文设置的数据集：

| 数据集类型 | 路径 | 特点 | 推荐程度 |
|------------|------|------|----------|
| **选定上下文** | `datasets/selective-context/` | 仅包含相关类的方法签名 | ⭐⭐⭐⭐⭐ 推荐 |
| 最大上下文 | `datasets/maximum-context/` | 包含整个项目骨架 | ⭐⭐⭐ |
| 最小上下文 | `datasets/minimum-context/` | 仅包含当前类 | ⭐⭐ |

## 二、数据集文件结构

### 2.1 文件命名

```
datasets/
├── selective-context/
│   ├── data-PA19.jsonl      # PA19 项目数据
│   ├── data-PA19-fix.jsonl  # PA19 修正版数据
│   ├── data-PA20.jsonl      # PA20 项目数据
│   ├── data-PA21.jsonl      # PA21 项目数据
│   └── data-PA22.jsonl      # PA22 项目数据
├── maximum-context/
│   └── [同上]
├── minimum-context/
│   └── [同上]
└── testcase/
    ├── test-PA19.jsonl      # PA19 测试配置
    ├── test-PA20.jsonl      # PA20 测试配置
    ├── test-PA21.jsonl      # PA21 测试配置
    └── test-PA22.jsonl      # PA22 测试配置
```

### 2.2 数据格式（JSONL）

每行一个 JSON 对象，格式如下：

```json
{
  "task_id": "PA19/Cell.java",
  "target": "game/map/cells/Cell.java",
  "code": "```java\n...\n```",
  "code_context": "```java\n...\n```"
}
```

## 三、数据字段详解

### 3.1 task_id（任务 ID）

**格式**：`{项目编号}/{类名}.java`

**示例**：
- `PA19/Cell.java`
- `PA20/Game.java`
- `PA21/Player.java`
- `PA22/Move.java`

**作用**：唯一标识一个代码生成任务

### 3.2 target（目标路径）

**格式**：相对路径，指向需要生成的类文件

**示例**：
- `game/map/cells/Cell.java`
- `assignment/protocol/Game.java`
- `pa1/model/Player.java`

**作用**：指示生成代码应该替换的位置

### 3.3 code（代码骨架）

**内容**：包含 TODO 标记的代码骨架

**示例**：
```java
package game.map.cells;

import game.MapElement;
import util.Coordinate;

/**
 * Representation of a cell in the {@link game.map.Map}.
 */
public abstract class Cell implements MapElement {

    public final Coordinate coord;

    /**
     * @param coord coordination of {@code this} cell
     */
    Cell(Coordinate coord) {
        this.coord = coord;
    }

    /**
     * Parses a {@link Cell} from a character.
     *
     * @param c Character to parse. For example, 'W' refers to a wall.
     * @param coord Coordinate of the newly created cell.
     * @param terminationType If the character is a termination cell, its type.
     * @return A cell based on the given creation parameters, or null if invalid.
     */
    public static Cell fromChar(char c, Coordinate coord, TerminationCell.Type terminationType) {
        // TODO

        return null;
    }
}
```

**特点**：
- 包含完整的类定义
- 方法签名和文档注释完整
- TODO 标记指示需要生成的部分
- 已实现的部分代码会保留

### 3.4 code_context（上下文信息）

**内容**：相关类的信息（根据上下文设置不同而不同）

#### 选定上下文（Selective Context）

**包含内容**：仅相关类的方法签名

**示例**：
```java
package game;

public interface MapElement {

    char toSingleChar() {
    }
}

package game.map.cells;

public class FillableCell extends Cell implements MapElement {

    private Pipe pipe;

    public FillableCell(Coordinate coord) {
    }

    public FillableCell(Coordinate coord, Pipe pipe) {
    }

    public Optional<Pipe> getPipe() {
    }

    @Override
    public char toSingleChar() {
    }

    public void setPipe(Pipe pipe) {
    }
}

package util;

public class Coordinate {

    public final int row;

    public final int col;

    public Coordinate(int row, int col) {
    }

    @Override
    public boolean equals(Object obj) {
    }

    public boolean equals(Coordinate other) {
    }

    @Override
    public int hashCode() {
    }

    public Coordinate add(Coordinate other) {
    }
}
```

**优势**：
- 信息精简，避免噪音
- 包含必要的依赖信息
- 减少 token 消耗

#### 最大上下文（Maximum Context）

**包含内容**：整个项目的所有类（受 LLM 上下文窗口限制）

**优势**：信息最全面  
**劣势**：
- Token 消耗大
- 可能超出上下文窗口
- 信息噪音多

#### 最小上下文（Minimum Context）

**包含内容**：仅当前类本身

**优势**：
- Token 消耗最少
- 适合快速测试

**劣势**：
- 缺少依赖信息
- 可能生成错误的代码

## 六、数据集使用示例

### 6.1 加载数据集

```python
import json

def load_dataset(file_path):
    """加载 JSONL 数据集"""
    data = []
    with open(file_path, 'r', encoding='utf-8') as f:
        for line in f:
            data.append(json.loads(line.strip()))
    return data

# 加载 PA19 的选定上下文数据集
dataset = load_dataset('datasets/selective-context/data-PA19.jsonl')

# 查看第一个任务
task = dataset[0]
print(f"Task ID: {task['task_id']}")
print(f"Target: {task['target']}")
print(f"Code length: {len(task['code'])} characters")
print(f"Context length: {len(task['code_context'])} characters")
```

### 6.2 构建 Prompt

```python
def build_prompt(code, code_context):
    """构建完整的 prompt"""
    prompt = f"""You are a Java programming assistant. 
Please complete the following Java class by implementing all methods marked with `// TODO`.

Here is the context information about related classes:

{code_context}

Now, please complete this class:

{code}

Requirements:
1. Implement all methods marked with `// TODO`
2. Follow the provided Javadoc comments
3. Ensure the code compiles successfully
4. Maintain proper OOP principles (encapsulation, inheritance, polymorphism)
5. Return the complete class implementation

Please provide the complete implementation:
"""
    return prompt

# 使用示例
task = dataset[0]
prompt = build_prompt(task['code'], task['code_context'])
```

### 6.3 提取生成代码

```python
import re

def extract_code(completion):
    """从 LLM 输出中提取 Java 代码"""
    # 匹配 ```java ... ``` 代码块
    pattern = r'```java\s*(.*?)\s*```'
    matches = re.findall(pattern, completion, re.DOTALL)
    
    if matches:
        return matches[0]
    
    # 如果没有代码块标记，尝试直接返回
    return completion.strip()

# 使用示例
llm_output = """
Here's the implementation:

```java
public class Cell {
    // implementation
}
```
"""

code = extract_code(llm_output)
```

    --model-path gpt-3.5-turbo \
    --num-sample 5 \

## 八、数据集质量分析

### 8.1 质量保证措施

1. **文档完整性**
   - 所有方法都有 Javadoc 注释
   - 参数和返回值说明清晰
   - 包含使用示例和注意事项

2. **代码骨架质量**
   - 类结构完整
   - 方法签名准确
   - 已实现部分经过验证

3. **上下文准确性**
   - 使用 jdeps 工具提取依赖
   - 人工验证依赖关系
   - 确保上下文信息完整

### 8.2 已知问题

1. **PA19 修正版**
   - `data-PA19-fix.jsonl` 修复了原始版本的一些问题
   - 建议使用修正版

2. **上下文窗口限制**
   - 最大上下文可能超出某些 LLM 的限制
   - 建议优先使用选定上下文

## 九、自定义数据集

### 9.1 创建自定义任务

```python
def create_custom_task(project_id, class_name, code, context):
    """创建自定义任务"""
    task = {
        "task_id": f"{project_id}/{class_name}.java",
        "target": code['path'],
        "code": f"```java\n{code['skeleton']}\n```",
        "code_context": f"```java\n{context}\n```"
    }
    return task

# 使用示例
custom_task = create_custom_task(
    project_id="PA19",
    class_name="CustomClass",
    code={
        "path": "game/CustomClass.java",
        "skeleton": "public class CustomClass { // TODO }"
    },
    context="public class RelatedClass { }"
)

# 保存到文件
with open('custom_task.jsonl', 'a') as f:
    f.write(json.dumps(custom_task) + '\n')
```
