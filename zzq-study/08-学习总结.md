## 二、关键洞察

### 2.1 关于第三方库

**重要发现**：
```
✅ JavaBench 核心代码不依赖任何第三方业务库
✅ 仅依赖测试框架（JUnit）和注解（JetBrains Annotations）
✅ 非常适合评估 LLM 的纯 Java 编程能力
```

**依赖清单**：
- JUnit Jupiter 5：测试框架
- JetBrains Annotations：@NotNull, @Nullable 等
- Mockito（仅 PA22）：Mocking 框架

**结论**：完全符合"不需要包含 Java 第三方库"的需求。

## 三、实践指南总结

### 3.1 快速上手路径

```
步骤 1：环境准备
├── 安装 Python 3.8+
├── 安装 Java 11+
└── pip install -r requirements.txt

步骤 2：数据理解
├── 查看 datasets/selective-context/
├── 理解 JSONL 格式
└── 阅读 task 结构

步骤 3：模型推理
├── 选择模型（如 gpt-3.5-turbo）
├── 运行 inference.py
└── 保存 samples.jsonl

步骤 4：评估分析
├── 运行 class-wise 评估
├── 运行 test-wise 评估
└── 分析结果指标
```

### 3.2 推荐配置

**标准评估配置**：
```yaml
数据集: datasets/selective-context/data-PA19.jsonl
策略: holistic
样本数: 5
温度: 0.2
最大 token: 4096
```

**最佳实践**：
- 使用 selective-context（效果最佳）
- 使用 holistic 策略（一次性生成）
- 生成至少 5 个样本（稳定性）
- 使用 test-wise 评估（准确性）


**结论**：JavaBench **完全符合**研究需求。

### 4.2 优势与局限

**优势**：
✅ 无第三方库依赖，评估纯粹  
✅ 项目级复杂度，接近真实场景  
✅ OOP 特性完整，深度评估  
✅ 质量保证充分，数据可靠  
✅ 工具完善，易于使用  
✅ 排行榜公开，便于对比  

**局限**：
⚠️ 仅 Java 语言，无法评估跨语言能力  
⚠️ 项目规模中等，不涉及大规模系统  
⚠️ 不包含并发、Web 等高级特性  
⚠️ 难度适中，对顶级模型挑战有限
